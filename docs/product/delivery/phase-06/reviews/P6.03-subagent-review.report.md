**Advisory Observations**

**`DailyTitleContent` rAF loop leaks on unmount.** `onMount` assigns `const animationFrame = requestAnimationFrame(updateTime)` and returns `() => cancelAnimationFrame(animationFrame)`. But `updateTime` schedules a new frame recursively via `requestAnimationFrame(updateTime)` on every call. The cleanup only cancels the initial frame handle, not any subsequently scheduled frame (the loop was already running by the time cleanup fires). The loop continues indefinitely after unmount until the browser garbage-collects the closure. This appears to be a pre-existing bug carried over unchanged from the Svelte 4 version (behavior-preserving refactor), so it is outside the three finding-discipline clauses, but it is real.

**`BreakdownChart` Svelte runtime warning for `state_referenced_locally`.** `let isFiltered = $state(_isFiltered)` will emit the Svelte 5 diagnostic `state_referenced_locally` at dev time. This is not a correctness gap (behavior is intentional), but it may produce noise in the dev console and could confuse future maintainers who encounter the warning without context. A suppression comment or a brief inline note would remove the ambiguity.

**`StackedBarChart` double-init on first render.** `onMount` initialises the chart and calls `setOption`; immediately afterward `$effect` fires, disposes that instance, and reinits. On a single mount this costs two `echarts.init` + two `setOption` calls. This pattern differs from `WeekdaysBarChart` (lazy-init, no dispose) and `BreakdownChart` (conditional init, no dispose) in the same ticket. The inconsistency is not a bug, but it means `StackedBarChart` does the most work on mount for no benefit. This is outside the three finding-discipline clauses.

**`createXAxisValues` return type diverges from `createStackedBarChartOption` parameter type.** `createXAxisValues` returns `string[] | undefined`; `createStackedBarChartOption` accepts `string[]`. TypeScript does not catch this because the assignment flows through `const xValues = $derived(createXAxisValues(summaries))` typed as the return type of the function.
