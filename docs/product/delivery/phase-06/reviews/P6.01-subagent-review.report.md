**Advisory Observations**

- **Double `setOption` on initial mount.** `onMount` calls `chart.setOption(option)` unconditionally (line 41), then `$effect` fires on its first run — after `onMount` but still on the same initial render — and calls `chart?.setOption(option)` again with the same object. The recipe in the implementation plan documents the pattern as "setOption once in onMount + $effect for subsequent updates," which could suggest the first `$effect` run is expected to be a no-op; it is not. If T03/T04 apply this recipe verbatim to 12 sibling charts, all of them will double-call on mount. The guard's actual safety window is only the pre-mount interval, which Svelte 5's `onMount`-before-effects ordering makes unreachable. The code is correct; the guard's meaning could mislead a future reader into believing it also protects the post-dispose case.
